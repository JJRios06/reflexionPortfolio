Reflexión sobre el Proyecto de Portfolio Web

1. Defensa del Proyecto

El objetivo principal de este proyecto ha sido el desarrollo de una página web para mostrar mi portafolio profesional. La intención es proporcionar una plataforma visualmente atractiva y funcional que permita a posibles empresas o clientes conocer mis habilidades y experiencia.

El diseño busca ser minimalista y moderno, garantizando una navegación intuitiva y una presentación clara de la información relevante. Se ha priorizado la compatibilidad con distintos dispositivos mediante un diseño responsivo.

2. Tecnologías Utilizadas

  • HTML5: Lenguaje base para la estructura del sitio.
  • CSS3: Definición de estilos para lograr una apariencia atractiva y moderna.
  • Bootstrap: Framework CSS que ha permitido agilizar el diseño responsivo, mejorar la estructura visual y aplicar componentes predefinidos.

Bootstrap ha sido una pieza clave en el desarrollo, ya que ha permitido simplificar la maquetación y mejorar la compatibilidad con diferentes dispositivos sin necesidad de escribir grandes cantidades de código CSS personalizado.

3. Aprendizaje Obtenido

Durante el desarrollo de este proyecto, he reforzado mis conocimientos en HTML, CSS y Bootstrap, mejorando mi capacidad para estructurar y estilizar una página web de manera eficiente. Además, he aprendido a gestionar mejor el uso de fuentes personalizadas y la organización de archivos en un proyecto web.

4. Problemas Encontrados y Soluciones

  • Problema con la alineación de elementos: Algunas secciones no se mostraban correctamente en dispositivos móviles.
  • Solución: Uso de las clases de Bootstrap para alineación y distribución de elementos.
  • Carga incorrecta de fuentes personalizadas: En algunos navegadores, las fuentes no se cargaban correctamente.
  • Solución: Ajuste en las rutas de los archivos y definición de varios formatos de fuente en la hoja de estilos CSS.

5. Accesibilidad y Usabilidad

Accesibilidad

  • Uso de etiquetas semánticas en HTML.
  • Inclusión de atributos alt en imágenes.
  • Contrastes adecuados entre texto y fondo.

Usabilidad

  • Diseño responsivo.
  • Menú de navegación claro.
  • Botones y enlaces accesibles.

6. Conclusiones y Posibles Mejoras

  • Implementación de JavaScript para mejorar la interactividad.
  • Inclusión de una sección dinámica con JSON o base de datos.
  • Optimización del rendimiento y carga de imágenes.

En general, el proyecto cumple con los objetivos propuestos y sienta una base sólida para futuras mejoras y expansiones.
